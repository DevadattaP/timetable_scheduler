About This System

Automated academic scheduling using Integer Linear Programming

🎓 Objective

This system automates the creation of a feasible and optimized class timetable for any academic programme. It accommodates multiple sections, faculty constraints, and calendar availability — while minimizing pedagogical disruptions.

⚙ How It Works
  • Configure sections, courses, faculty, mappings, and constraints in the Configure tab.
  • You can import configuration from an Excel file. Download Template
  • Click Generate Timetable to send the configuration to the LP solver.
  • The solver always tries to satisfy all enabled Hard Constraints.
  • If the Consecutive Sessions Rule is enabled, the solver minimizes consecutive-period violations using a Soft Penalty Objective.
  • The generated timetable can be verified against all configured constraints.
  • You can export both configuration and timetable to Excel for backup or offline editing.
🔒 Hard Constraints
  • Zero Slack: Every available slot must be filled by exactly one class.
  • No Faculty Cloning: A faculty member cannot teach two sections simultaneously.
  • Max Daily Load: Faculty can teach at most N sessions per calendar day.
  • Course Spacing: A section cannot have the same course twice in one day.
  • Unavailability (optional): Faculty are not scheduled on blocked dates.
  • Course conflicts (optional): Courses in specified groups cannot be scheduled at the same time.
✨ Soft Constraint — Consecutive Sessions Rule
  • Configurable Consecutive Limit: Restrict how many consecutive periods the same course may appear for a section.
  • Flexible Period Unit: Consecutive tracking can be applied across weeks or days.
  • Reset Boundary: Consecutive counting may optionally reset at boundaries such as month, or remain continuous throughout the schedule.
  • Penalty-Based Optimization: Violations are treated as soft penalties in the LP objective function, while all hard constraints remain strictly enforced.
  • Violation Definition: A penalty is added whenever the same course appears in (max consecutive + 1) consecutive eligible periods for the same section.
  • Example 1: If Max Consecutive = 2 Weeks with Month Boundary, a penalty is triggered when the same course runs for 3+ consecutive weeks within the same month.
  • Example 2: If Max Consecutive = 1 Week with No Boundary, it works like an alternate week rule, and a penalty is triggered when the same course runs for 2+ consecutive weeks within the complete schedule.

🔒 Fixed Constraints Always Applied
  • H1 – Exact Fulfillment: Every section completes exactly the required number of sessions per course.
  • H2 – One Course per Slot: Every available slot is filled by exactly one course (zero slack).
  • H3 – No Faculty Cloning: A faculty member cannot teach two sections simultaneously.
  • H4 – Max Daily Workload: Faculty cannot exceed their configured maximum sessions per calendar day.
  • H5 – Daily Course Spacing: A section cannot have the same course more than once in a single day.
⚙ Optional Constraints

A penalty is added for each window where the same course appears in (max + 1) consecutive periods. Solver minimizes total penalty.
Example: Max 2 weeks / Month boundary → penalty fires if 3+ consecutive Saturdays in the same month cover the same course.
If you want alternate week rule, set Max 1 week with no boundary.
