About This System

Automated academic scheduling using Integer Linear Programming

🎓 Objective

This system automates the creation of a feasible and optimized class timetable for any academic programme. It accommodates multiple sections, faculty constraints, and calendar availability — while minimizing pedagogical disruptions.

⚙ How It Works
  • Configure sections, courses, faculty, and their mappings in the Configure tab.
  • You can even import configuration from an Excel file. Download Template
  • Click Generate Timetable to send data to the LP solver.
  • The solver first attempts a Hard model (no consecutive-week violations).
  • If infeasible, it automatically retries with a Soft penalty model.
  • The generated timetable is displayed and can be verified for all constraint satisfaction.
  • You can export the configuration and timetable to Excel for backup or offline editing.
🔒 Hard Constraints
  • Zero Slack: Every available slot must be filled by exactly one class.
  • No Faculty Cloning: A faculty member cannot teach two sections simultaneously.
  • Max Daily Load: Faculty can teach at most N sessions per calendar day.
  • Course Spacing: A section cannot have the same course twice in one day.
  • Unavailability: Faculty are not scheduled on blocked dates.
✨ Soft Constraint (Optimized)
  • Alternate Weekend Rule: A section should not be taught the same course in two consecutive weeks.
  • If a perfect schedule is impossible (due to zero slack), violations are minimized via a penalty in the objective function.
  • Penalty = number of (section, course, week) pairs where the same course appears in both week W and W+1.
